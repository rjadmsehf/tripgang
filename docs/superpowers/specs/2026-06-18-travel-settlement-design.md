# 여행 정산 앱 (Travel Settlement Web App) - Design Specification

본 문서는 실시간 환율 정보와 Gemini Vision API를 결합하여 여행지에서 촬영한 외화 영수증을 자동으로 인식·번역하고, 동행자별로 메뉴 단위 정산을 지원하는 웹 애플리케이션의 디자인 규격서입니다.

---

## 1. 아키텍처 및 기술 스택
* **프론트엔드 프레임워크**: React (Vite 기반 빌드)
* **스타일링**: Vanilla CSS (글래스모피즘 및 다크 모드 테마, 프리미엄 UI 디자인 적용)
* **LLM 영수증 분석**: `@google/generative-ai` SDK를 이용한 클라이언트 사이드 Gemini 3.5 Flash API 연동
* **환율 정보**: 무료 환율 API (`https://open.er-api.com/v6/latest/USD` 또는 유사) 연동
* **상태 관리**: React 내장 State 및 `localStorage` (API Key 및 정산 내역 세션 유지용)

---

## 2. 핵심 기능 요구사항 및 데이터 흐름

### A. 동행자 및 설정 관리
* **API Key 설정**: 사용자가 입력한 Gemini API Key를 `localStorage`에 저장하여 사용.
* **동행자 리스트**: 이름을 입력해 동행자(멤버)를 유동적으로 추가 및 삭제 가능.
* **개별 내역서 생성**: 동행자가 추가되면 자동으로 해당 멤버의 정산 카드가 생성됨.

### B. 영수증 OCR & 번역 (Gemini API 연동)
* **업로드**: 영수증 파일 드래그 앤 드롭 또는 파일 선택기.
* **LLM 분석**: 업로드된 이미지를 base64로 변환하여 Gemini 3.5 Flash 모델에 전송.
* **엄격한 JSON 스키마 응답 유도**:
  ```json
  {
    "currency": "JPY", // 영수증에 표시된 통화 기호 (USD, JPY, CNY, EUR, KRW 등)
    "totalAmount": 4500, // 영수증 총액
    "items": [
      {
        "name": "醤油ラーメン", // 영수증 원문 메뉴명
        "translatedName": "소유 라멘", // 한국어 번역 메뉴명
        "price": 950 // 해당 메뉴 단일/총 가격
      }
    ]
  }
  ```

### C. 실시간 환율 변환
* API를 통해 실시간으로 `Original Currency`에서 `KRW`로 환율을 가져옴.
* 영수증 분석 결과 통화(예: JPY)에 맞추어 메뉴 가격 및 총액을 원화(KRW)로 실시간 환산하여 표시.
* 환율 정보를 사용자가 수동으로 미세 조정할 수 있는 입력 필드도 제공.

### D. 메뉴 할당 및 정산 (N분의 1 지원)
* 분석된 각 메뉴 항목 옆에 동행자들의 이름이 체크박스 형태로 표시됨.
* 체크박스를 클릭하여 해당 메뉴를 먹은 사람들을 지정:
  * 예: 900엔짜리 소유 라멘을 `철수`와 `영희`가 체크하면, 각각 450엔(원화 환산 약 4,000원)씩 내역서에 자동 추가.
* 여러 개의 영수증을 누적해서 하나의 정산 세션에 등록할 수 있음.

---

## 3. UI/UX 디자인 구성 (방안 A: 단일 화면 대시보드)

### 전체적인 비주얼 테마
* **테마**: 심해 블루 및 네온 퍼플 포인트의 글래스모피즘(Glassmorphic) 다크 모드.
* **인터랙션**: 업로드 시 펄스 애니메이션, 영수증 파싱 성공 시 다이내믹 피드백, 멤버 카드 호버 효과.

### 레이아웃 구조 (3단 반응형 그리드)
1. **좌측 패널 (설정 및 영수증 업로드)**
   * Gemini API Key 입력 창 (마스킹 처리).
   * 여행 멤버 관리 창 (입력창 + 멤버 태그 리스트).
   * 드래그 앤 드롭 영수증 업로더 영역.
2. **중앙 패널 (영수증 분석 및 메뉴 배분)**
   * 업로드된 영수증 정보 (통화 종류, 환율 정보 표시).
   * 메뉴 아이템 테이블: [원문 메뉴명] / [번역명] / [외화 단가] / [원화 환산가] / [먹은 사람 지정 (체크박스 목록)].
   * 수동 아이템 추가 버튼 (OCR이 누락되거나 영수증 없이 직접 정산할 때 사용).
3. **우측 패널 (동행자별 청구서 & 송금 요약)**
   * 동행자별 정산 카드: 멤버명, 소비한 총액(외화/원화), 소비한 세부 메뉴 목록.
   * 최종 정산 결과 요약: "총 사용 금액 및 멤버별 송금해야 할 최종 금액" 표시.

---

## 4. 예외 및 오류 처리
* **API Key 미입력**: 영수증 업로드 시 API Key 입력을 요청하는 안내 모달 표시.
* **환율 API 호출 실패**: 기본 환율값(엔화 100엔 = 900원, 위안화 1위안 = 190원 등)으로 폴백 작동 후 수동 조절 가능하게 처리.
* **Gemini 파싱 에러**: 영수증 이미지가 흐리거나 형식이 다를 경우, 분석 결과를 직접 수동으로 추가 및 편집할 수 있도록 에디터 제공.

---

## 5. 검증 계획
* **수동 검증**: 
  * 일본 엔화 영수증, 중국 위안화 영수증 이미지를 업로드하여 올바르게 통화와 가격이 파싱되고 환율 변환이 되는지 확인.
  * 여러 멤버를 지정했을 때 분할된 금액이 정확하게 소수점 아래에서 절사 또는 반올림되어 원화로 계산되는지 체크.
